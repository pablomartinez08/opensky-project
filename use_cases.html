UC-01

Alerta Vertical — Ascenso / Descenso Brusco

Detecta cuando una aeronave experimenta un cambio de altitud anómalo en un periodo corto de tiempo. Flink mantiene un ValueState con el historial reciente de altitudes por cada icao24. Si la diferencia entre la altitud actual y la de hace N segundos supera un umbral definido (ej. ±2.000 metros en 30 segundos), se genera una alerta de severidad proporcional al delta.

Ejemplo real: Un avión desciende de 11.200m a 8.900m en 20 segundos — posible emergencia, turbulencia severa o evasión de colisión (TCAS RA).

UC-02

Alta Densidad — Concentración de Tráfico en Celda H3

Cada posición GPS se indexa en una celda hexagonal H3 (resolución 4–5). Flink cuenta el número de aeronaves dentro de cada celda en una ventana tumbling de 5 minutos. Si en esa ventana se acumulan más de X aeronaves en la misma celda H3, se genera una alerta de alta densidad que indica una concentración inusual de tráfico aéreo.

Ejemplo real: La celda H3 86395b467ffffff (zona centro de Madrid) registra 85 aeronaves en una ventana de 5 minutos — posible congestión, evento especial o condiciones meteorológicas que obligan a concentrar rutas.

UC-03

Alerta de Zona Muerta — Ausencia de Tráfico en Celda H3

Cada posición GPS se indexa en una celda hexagonal H3 (resolución 4–5, ~22 km de diámetro). Flink mantiene un registro histórico de qué celdas suelen tener tráfico. Si una celda que habitualmente registra actividad aparece completamente vacía durante una ventana tumbling de 5 minutos, se genera una alerta de zona muerta.

Ejemplo real: La celda H3 841f91fffffffff (corredor aéreo sobre los Pirineos) normalmente registra 8–12 aeronaves cada 5 minutos. Durante la última ventana: 0 aeronaves. Posible restricción temporal del espacio aéreo, cierre militar o fallo de radar.

UC-04

Flink CEP — Detección de Patrones Complejos Encadenados

Utiliza la API de Flink CEP (Complex Event Processing) para definir patrones encadenados que sería imposible detectar con una sola ventana. La librería permite definir secuencias de condiciones con restricciones temporales (ej. "evento A seguido de evento B en menos de 2 minutos").

Ejemplo real: Un avión ejecuta un descenso brusco (−2000m) seguido inmediatamente de velocidad horizontal = 0 durante más de 60 segundos → posible aterrizaje de emergencia o accidente. Este patrón combina datos de los nodos UC-01 y UC-02 encadenados temporalmente.

Flink CEP Pattern API within(2 min) flight-alerts

    AlertaCritica --> [*]: Emitir alerta CEP

    DescensoBrusco --> Monitorizando: timeout 2min
    VelocidadCero --> Monitorizando: vel > 0 (recupera)
